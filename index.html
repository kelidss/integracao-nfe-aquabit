Documentação Técnica: Integração de Nota Fiscal Eletrônica (NFe) no Sistema Aquabit

Versão: 1.0 | Data: 26 de Setembro de 2025

1. Visão Geral

Este documento detalha a arquitetura e o plano de implementação para a integração de sistemas de emissão de Nota Fiscal Eletrônica (NFe) na plataforma Aquabit. O objetivo é automatizar e gerenciar o ciclo de vida das notas fiscais geradas a partir de vendas no sistema.

1.1. Objetivos Principais

  • Automatizar a Emissão: Gerar NFes automaticamente para todas as vendas concluídas na plataforma.
  • Múltiplos Provedores: Garantir flexibilidade através da integração com diversos provedores de NFe, incluindo Nuvem Fiscal, Focus NFe e Revgás.
  • Gestão Completa: Permitir a consulta de status, listagem e o download do Documento Auxiliar da Nota Fiscal Eletrônica (DANFE) e do arquivo XML.
  • Rastreabilidade: Manter um histórico detalhado e logs completos de todas as transações fiscais para auditoria e controle.

1.2. Tabelas Base de Dados

A integração utilizará as seguintes tabelas do banco de dados como fonte de informação para a geração das notas fiscais:

Tabela Descrição
core_movimentacao Armazena as movimentações financeiras gerais das vendas.
core_movimentacaoitem Detalha os itens específicos de cada movimentação.
core_venda Contém os dados específicos e consolidados de cada venda.
core_lotetanquevenda Associa os produtos vendidos aos seus respectivos lotes e tanques.

2. Arquitetura da Solução

A solução foi desenhada de forma modular, utilizando uma arquitetura baseada em Models, Services e Adapters para garantir escalabilidade e fácil manutenção.

2.1. Estrutura de Models

Os models abaixo representam a estrutura de dados no banco de dados para gerenciar provedores, configurações, notas fiscais e logs.

# apps/integracao/models.py

class ProvedorNFe(BaseClass):
    """
    Representa os provedores de NFe disponíveis no sistema.
    """
    nome = models.CharField(max_length=100)
    slug = models.SlugField(unique=True)
    api_url = models.URLField()
    ativo = models.BooleanField(default=True)

class ConfiguracaoNFe(BaseClass):
    """
    Armazena as configurações de NFe para cada propriedade (cliente),
    incluindo credenciais e ambiente de operação.
    """
    propriedade = models.ForeignKey('core.Propriedade', on_delete=models.CASCADE)
    provedor = models.ForeignKey(ProvedorNFe, on_delete=models.CASCADE)
    api_token = models.CharField(max_length=255)
    ambiente = models.CharField(max_length=20)  # 'homologacao' ou 'producao'
    certificado_digital = models.TextField(blank=True)
    senha_certificado = models.CharField(max_length=100, blank=True)

class NotaFiscal(BaseClass):
    """
    Registra cada NFe emitida, seu status, chave de acesso e
    armazena os arquivos XML e DANFE.
    """
    venda = models.ForeignKey('core.Venda', on_delete=models.CASCADE)
    provedor = models.ForeignKey(ProvedorNFe, on_delete=models.CASCADE)
    numero_nota = models.CharField(max_length=50)
    serie = models.CharField(max_length=10)
    chave_acesso = models.CharField(max_length=44)
    status = models.CharField(max_length=50)
    data_emissao = models.DateTimeField()
    xml_content = models.TextField(blank=True)
    pdf_danfe = models.FileField(upload_to='nfe/danfe/', blank=True)
    protocolo_autorizacao = models.CharField(max_length=100, blank=True)

class LogNFe(BaseClass):
    """
    Mantém um log detalhado de todas as operações relacionadas a uma NFe
    para fins de depuração e auditoria.
    """
    nota_fiscal = models.ForeignKey(NotaFiscal, on_delete=models.CASCADE)
    operacao = models.CharField(max_length=50)  # 'emissao', 'consulta', 'cancelamento'
    status_code = models.IntegerField()
    resposta = models.TextField()
    erro = models.TextField(blank=True)

2.2. Camada de Serviço (Services)

O NFEService atua como a camada central de lógica de negócios, orquestrando as operações de NFe e utilizando um padrão de Factory para selecionar o Adapter correto para cada provedor.

# apps/integracao/services/nfe_service.py

class NFEService:
    """
    Serviço base para orquestrar as operações de NFe.
    """

    def __init__(self, configuracao: ConfiguracaoNFe):
        self.configuracao = configuracao
        self.adapter = self._get_adapter()

    def _get_adapter(self):
        """
        Factory Method que instancia o adapter correto com base
        no provedor configurado.
        """
        adapters = {
            'nuvem-fiscal': NuvemFiscalAdapter,
            'focus-nfe': FocusNFeAdapter,
            'revgas': RevgasAdapter,
        }
        adapter_class = adapters.get(self.configuracao.provedor.slug)
        if not adapter_class:
            raise ValueError("Provedor de NFe não suportado.")
        return adapter_class(self.configuracao)

    def emitir_nfe(self, venda: Venda) -> NotaFiscal:
        """
        Coordena o processo de emissão de uma NFe para uma venda específica.
        """
        # ...lógica de negócio...
        return self.adapter.emitir(venda)

    def consultar_nfe(self, nota_fiscal: NotaFiscal) -> dict:
        """
        Consulta o status atual de uma NFe já emitida.
        """
        # ...lógica de negócio...
        return self.adapter.consultar(nota_fiscal)

    def baixar_danfe(self, nota_fiscal: NotaFiscal) -> bytes:
        """
        Realiza o download do PDF da DANFE.
        """
        # ...lógica de negócio...
        return self.adapter.baixar_pdf(nota_fiscal)

    def baixar_xml(self, nota_fiscal: NotaFiscal) -> bytes:
        """
        Realiza o download do arquivo XML da NFe.
        """
        # ...lógica de negócio...
        return self.adapter.baixar_xml(nota_fiscal)

4. Endpoints da API

A gestão da integração será realizada através dos seguintes endpoints na API v2.

4.1. Notas Fiscais (/api/v2/nfe/)

Método Endpoint Descrição
GET / Lista todas as NFes emitidas com filtros.
POST / Cria um registro de Nota Fiscal (preparação).
GET /{id}/ Detalha uma NFe específica.
PUT /{id}/ Atualiza os dados de uma NFe (uso restrito).
DELETE /{id}/ Remove um registro de NFe (cancelamento lógico).
POST /{id}/emitir/ Dispara o processo de emissão da NFe.
GET /{id}/consultar/ Consulta o status mais recente da NFe no provedor.
GET /{id}/baixar_danfe/ Realiza o download do arquivo PDF da DANFE.
GET /{id}/baixar_xml/ Realiza o download do arquivo XML autorizado.

4.2. Configurações (/api/v2/configuracao-nfe/)

Método Endpoint Descrição
GET / Lista todas as configurações de NFe.
POST / Cria uma nova configuração para uma propriedade.
GET /{id}/ Detalha uma configuração específica.
PUT /{id}/ Atualiza uma configuração existente.
DELETE /{id}/ Remove uma configuração.

5. Deployment e Configuração

5.1. Variáveis de Ambiente

As seguintes variáveis de ambiente devem ser configuradas para o correto funcionamento da integração:

Variável Descrição Exemplo
NFE_DEFAULT_PROVIDER Provedor padrão a ser utilizado. nuvem-fiscal
NFE_TIMEOUT Timeout em segundos para as requisições à API. 30
NFE_MAX_RETRIES Número máximo de tentativas em caso de falha. 3
NFE_RETRY_DELAY Atraso em segundos entre as tentativas. 5
NUVEM_FISCAL_BASE_URL URL base da API da Nuvem Fiscal. https://api.nuvemfiscal.com.br
FOCUS_NFE_BASE_URL URL base da API da Focus NFe. https://api.focusnfe.com.br
REVGAS_BASE_URL URL base da API da Revgás. https://revgas.stoplight.io

5.2. Configuração de Filas (Celery)

Para garantir que a emissão de notas fiscais não impacte a performance da aplicação principal, as operações serão processadas de forma assíncrona utilizando Celery. As tarefas serão roteadas para uma fila dedicada chamada nfe.

# settings.py

CELERY_ROUTES = {
    'apps.integracao.tasks.emitir_nfe': {'queue': 'nfe'},
    'apps.integracao.tasks.consultar_status_nfe': {'queue': 'nfe'},
}
